# 🦈 分批次持倉系統使用指南

## 🎯 功能概述

現在系統支援**分批次持倉顯示**！

### 主要特色

1. ✅ **每次買入獨立記錄** - 今天買 1000 股，明天再買 1000 股，會分兩行顯示
2. ✅ **顯示買入日期** - 每批次都有明確的買入日期
3. ✅ **計算持有天數** - 自動計算從買入至今的天數
4. ✅ **按股票代號排序** - 同一股票的不同批次會排在一起
5. ✅ **FIFO 賣出** - 賣出時會優先賣出最早買入的批次

---

## 📊 持倉明細新增欄位

| 欄位 | 說明 |
|------|------|
| 代號 | 股票代號 |

| **買入日期** | 這批次的買入日期 (新！) |
| **持有天數** | 從買入至今的天數 (新！) |
| 建倉理由 | 買入時的備註 |
| 操作 | 賣出按鈕 |

**剩餘**：
- 只剩下 2026/3/27 買的那批
- 還沒賣，繼續持有

## 💡 為什麼要分批次顯示？

### 優點

1. **清楚追蹤成本** - 知道每批的實際成本
2. **檢討策略** - 看看哪次買入時機最好
3. **稅務規劃** - 未來實際投資時，可選擇賣出特定批次（節稅）
4. **心理建設** - 看到不同批次的表現，學習市場波動

### 實際應用

**情境：短期獲利 vs 長期持有**

```
第1批（長期持有）：
2026/1/10 買入 1000股 @ $25.00
持有 84 天，現價 $30.00，獲利 +20%

第2批（短期操作）：
2026/3/20 買入 1000股 @ $28.50
持有 12 天，現價 $30.00，獲利 +5.26%
```

**決策參考**：
- 如果要獲利了結，可以賣第2批（短期）
- 第1批繼續持有，享受長期複利

## 🎯 常見問題

### Q: 如果我賣出部分股數，會賣哪一批？

A: 系統使用 **FIFO（先進先出）** 原則，會優先賣出最早買入的批次。

**範例：**
```
持有：
- 2026/3/25 買 1000股 @ $25
- 2026/3/30 買 1000股 @ $27

賣出 500股 → 會從 2026/3/25 那批賣 500 股

剩餘：
- 2026/3/25 買 500股 @ $25
- 2026/3/30 買 1000股 @ $27
```

### Q: 如果我想合併顯示同一股票，怎麼辦？

A: 目前系統預設是分批次顯示。如果你想看「合併後」的總持股，可以：
1. 查看「總覽」區域（會顯示總成本、總市值）
2. 或者使用舊版 `GET /api/holdings` API（會合併同一股票）

### Q: 持有天數是怎麼計算的？

A: `持有天數 = 今天日期 - 買入日期`（以天為單位）

### Q: 如果即時股價讀取失敗會怎樣？

A: 系統會自動降級到舊版 `portfolio.json`，或使用買入價作為現價（損益 = 0）。
